ZettaPay is the payment layer for the agentic AI economy. Every major
agent runtime in production today already speaks one of two protocols
that ZettaPay implements natively:

- **MCP** — for tool-calling agents (Claude, ChatGPT operators, custom
  hosts that embed `@modelcontextprotocol/sdk`).
- **x402** — for HTTP-driven agents that pay inline via a header (any
  agent that can `fetch`, including Hugging Face Inference clients,
  LangChain tools, and bare cURL).

Because these are open specs, no formal partnership is needed for an
agent to transact. The integrations below are the supported, tested
recipes — pick the runtime you ship on and copy the snippet.

## Compatibility matrix

| Runtime | Protocol | Auth method | Status |
| --- | --- | --- | --- |
| Claude Desktop | MCP over HTTPS | `x-zettapay-api-key` header | Supported |
| Claude API (`tool_use`) | MCP via host bridge | Host-managed key | Supported |
| OpenAI Assistants v2 | x402 + function tool | `x-zettapay-api-key` header | Supported |
| OpenAI function calling | x402 + function tool | `x-zettapay-api-key` header | Supported |
| Hugging Face Inference | x402 header | `x-zettapay-api-key` header | Supported |
| `smolagents` / transformers | x402 header | `x-zettapay-api-key` header | Supported |
| LangChain / LangGraph | x402 + tool wrapper | `x-zettapay-api-key` header | Supported |

ZettaPay does not custody any keys for any of the runtimes above. The
host application holds the merchant API key; the agent's signing key
is scoped to a single transaction at signing time. See
[AI agents](/concepts/ai-agents) for the full security model.

## Why native, not branded

ZettaPay deliberately ships against open protocols rather than
private SDKs from any single AI vendor:

- **No vendor lock-in.** The same payment call works whether the
  agent runtime is Claude, GPT, an open-weights model on Hugging
  Face, or a future provider that has not shipped yet.
- **Stable ABI.** MCP and x402 are standardized; vendor-specific
  function-calling shapes change between minor releases.
- **No custody handoff.** Branded SDKs typically expect to manage
  funds. ZettaPay never custodies USDC — the agent signs, ZettaPay
  submits. That property is preserved across every runtime.
