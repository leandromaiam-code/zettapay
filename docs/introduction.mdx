ZettaPay is an open payment protocol on Solana. It lets merchants accept
USDC with one HTTP call, lets AI agents pay autonomously through the
[x402](/guides/x402-protocol) header and [MCP](/guides/mcp-integration)
tools, and settles end-to-end in seconds at **0.30%** per transaction.

## Core primitives

| Primitive | What it does |
| --- | --- |
| **Merchants** | Wallet-bound identities that receive USDC. Created via `POST /merchants/register`. |
| **Payments** | One-shot `transferChecked` USDC transactions. Created via `POST /pay`. |
| **Subscriptions** | Recurring billing on a daily / weekly / monthly cadence. |
| **Settlement** | Optional fiat off-ramp via Coinflow. |
| **Webhooks** | HMAC-SHA256 signed events with up to 10 retry attempts. |
| **MoonPay onramp** | Card → USDC funding flow for paying customers. |

## Core constraints

ZettaPay V1 is intentionally narrow so the surface area stays auditable:

- **Solana mainnet / devnet only.** Multi-chain is on the post-PMF roadmap.
- **USDC only.** Other stablecoins land after PMF.
- **Non-custodial.** USDC always moves payer → merchant directly. ZettaPay
  never holds funds.
- **TLS-only webhooks.** HTTP webhook URLs are rejected at registration.
