Quickstart · 5 min

Accept your first USDC payment in five minutes.

From zero to live merchant on Solana devnet — no smart-contract experience required. Pick your stack, copy the snippet, run it. Receipts confirm in under a second.

TypeScript Python Go cURL

Overview

ZettaPay is a Solana-native payment protocol. Merchants register a wallet, payers sign a standard SPL token transfer, and the API submits it via the open x-402-payment header. There is no escrow — funds move payer → merchant directly.

Devnet only. This guide targets Solana devnet (USDC mint 4zMMC9srt5Ri5X14GAgXhaHii3GnPAEERYPJgZJDncDU). Mainnet ships post-audit.

Prerequisites

  • › Node 18+, Python 3.10+, or Go 1.22+ (pick one)
  • › A Solana wallet (Phantom recommended) on devnet
  • › Some devnet SOL — request from faucet.solana.com
  • › A ZettaPay API base URL — defaults to https://api.zettapay.dev

Pick the language you ship in.

// Native Go SDK — zero third-party dependencies

Register a merchant

Bind your wallet on-chain via the memo program. The protocol pays the rent (~0.002 SOL) for your USDC associated token account. Save the returned apiKey — you'll need it next. 

import { ZettaPayClient } from '@zettapay/sdk';

const client = new ZettaPayClient({
  baseURL: 'https://api.zettapay.dev',
});

const merchant = await client.registerMerchant({
  name: 'Acme Coffee',
  email: 'owner@acme.coffee',
  walletAddress: '7Np41oeYqPefeNQEHSv1UDhYrehxin3NStpSyab9YVhT',
});

console.log(merchant.id, merchant.apiKey);
import requests

resp = requests.post(
    "https://api.zettapay.dev/merchants/register",
    json={
        "name": "Acme Coffee",
        "email": "owner@acme.coffee",
        "walletAddress": "7Np41oeYqPefeNQEHSv1UDhYrehxin3NStpSyab9YVhT",
    },
    timeout=15,
)
resp.raise_for_status()
merchant = resp.json()
print(merchant["merchant"]["id"], merchant["apiKey"])
package main

import (
    "context"
    "log"
    "time"

    zettapay "github.com/leandromaiam-code/zettapay/packages/sdk-go"
)

func main() {
    client, _ := zettapay.NewClient(zettapay.ClientConfig{
        BaseURL: "https://api.zettapay.dev",
        Retry:   zettapay.DefaultRetryPolicy(),
    })

    ctx, cancel := context.WithTimeout(context.Background(), 15*time.Second)
    defer cancel()

    merchant, err := client.RegisterMerchant(ctx, zettapay.RegisterMerchantInput{
        Name:          "Acme Coffee",
        Email:         "owner@acme.coffee",
        WalletAddress: "7Np41oeYqPefeNQEHSv1UDhYrehxin3NStpSyab9YVhT",
    })
    if err != nil { log.Fatal(err) }
    log.Println(merchant.ID, merchant.APIKey)
}
curl -X POST https://api.zettapay.dev/merchants/register \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "name": "Acme Coffee",
    "email": "owner@acme.coffee",
    "walletAddress": "7Np41oeYqPefeNQEHSv1UDhYrehxin3NStpSyab9YVhT"
  }'
Idempotency. Every write endpoint accepts an Idempotency-Key header. The TypeScript and Go SDKs send one automatically; for Python and cURL pass any UUID v4.

Build a signed payment

ZettaPay never custodies funds. The payer signs a standard SPL transfer transaction client-side; you then base64-encode the signed bytes and forward them to the API. 

import {
  Connection, PublicKey, Transaction, clusterApiUrl,
} from '@solana/web3.js';
import {
  createTransferCheckedInstruction, getAssociatedTokenAddress,
} from '@solana/spl-token';

const connection = new Connection(clusterApiUrl('devnet'));
const usdcMint = new PublicKey('4zMMC9srt5Ri5X14GAgXhaHii3GnPAEERYPJgZJDncDU');

const payerAta    = await getAssociatedTokenAddress(usdcMint, payer.publicKey);
const merchantAta = await getAssociatedTokenAddress(usdcMint, merchantPubkey);

const ix = createTransferCheckedInstruction(
  payerAta, usdcMint, merchantAta, payer.publicKey,
  1_500_000,   // 1.50 USDC (6 decimals)
  6,
);

const tx = new Transaction().add(ix);
tx.feePayer = payer.publicKey;
tx.recentBlockhash = (await connection.getLatestBlockhash()).blockhash;
tx.sign(payer);

const signedBase64 = tx.serialize().toString('base64');
import base64

Submit via the x402 header

ZettaPay implements the open x402 spec: the signed transaction travels in the x-402-payment header, base64-encoded. The API broadcasts, confirms, and returns a typed receipt. 

const receipt = await client.pay({ transaction: signedBase64 });

console.log(receipt.paymentId);   // pay_01HM...
console.log(receipt.status);      // "confirmed"
console.log(receipt.signature);   // on-chain tx signature
resp = requests.post(
    "https://api.zettapay.dev/pay",
    headers={
        "x-402-payment": signed_b64,
        "Idempotency-Key": str(uuid.uuid4()),
    },
    timeout=30,
)
resp.raise_for_status()
receipt = resp.json()
print(receipt["paymentId"], receipt["status"])
receipt, err := client.PayBase64(ctx, signedB64)
if err != nil { log.Fatal(err) }

log.Println(receipt.PaymentID, receipt.Status, receipt.Signature)
curl -X POST https://api.zettapay.dev/pay \
  -H "x-402-payment: $SIGNED_TX_BASE64" \
  -H "Idempotency-Key: $(uuidgen)"

Verify the receipt

Receipts are immutable and queryable. Confirmation typically lands in under one second on devnet — verify with the payment id from step 4. 

const record = await client.getPayment(receipt.paymentId);
console.log(record.status, record.amount, record.signature);
payment = requests.get(
    f"https://api.zettapay.dev/payments/{receipt['paymentId']}",
    timeout=10,
).json()

print(payment["status"], payment["amount"])
payment, err := client.GetPayment(ctx, receipt.PaymentID)
if err != nil { log.Fatal(err) }
log.Println(payment.Status, payment.Amount, payment.Signature)
curl https://api.zettapay.dev/payments/pay_01HM...
Webhooks. For production, subscribe to the payment.confirmed event instead of polling. Signatures are HMAC-SHA256 over the body — see the webhook reference.

API reference (excerpt)

MethodPathPurpose
POST/merchants/registerBind wallet on-chain, create USDC ATA, return API key.
POST/paySubmit signed tx via x-402-payment header.
GET/payments/:idFetch a recorded payment by id.
GET/paymentsPaginated payment list (filter by merchant).
POST/webhooksRegister a TLS endpoint for payment.confirmed.
GET/healthzLiveness probe (200 = OK).

All write endpoints accept an Idempotency-Key header. Rate limits are per API key (sliding window). Errors mirror { code, message, details }.
