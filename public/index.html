How it works

Built for three kinds of payers

One protocol. Three integrations. Whether your customer is human, autonomous, or programmatic — ZettaPay just works.

AI Agents

Claude, GPT, Gemini. Pay autonomously via signed transactions in HTTP headers. No keys to custody.

# x402 header with signed Solana tx
POST /pay
x-402-payment: "<signed-tx-blob>"

{
  "merchant": "@coffee-bot",
  "amount_usdc": 2.50
}

Humans

Phantom wallet binding. Seamless UX. Card → USDC via MoonPay onramp built-in. No crypto knowledge required.

// Connect Phantom & pay USDC
const { tx } = await zettapay.pay({
  merchant: "@store",
  amount: 42.00,
  source: "phantom"
});

Merchants

Drop-in SDK. Production-grade webhooks with HMAC signing. Three-line integration. Accept payments in minutes.

import { ZettaPay } from '@zettapay/sdk';

const zp = new ZettaPay(apiKey);
await zp.merchants.register({
  name: "My Store"
});

x402-payment.ts 
// x402 Payment Header — agent pays autonomously
const response = await fetch("https://api.merchant.com/buy", {
  method: "POST",
  headers: {
    "x-402-payment": signedSolanaTx,
    "x-402-amount": "2.50",
    "x-402-currency": "USDC",
  },
  body: JSON.stringify({
    item: "cappuccino",
    merchant: "@cafe-bot"
  })
});
// -> 200 OK, settled in 1.4s on Solana

SDK

Three lines. One payment.

TypeScript SDK with full type safety. npm install and ship.

1. Install
$ npm install @zettapay/sdk
2. Initialize
import { ZettaPay } from '@zettapay/sdk';
const zp = new ZettaPay('sk_live_...');
3. Charge
const payment = await zp.pay({
  merchant: "@my-store",
  amount_usdc: 42.00,
  webhook_url: "https://my-app.com/zp-webhook",
  metadata: { order_id: "ord_xyz" }
});

// payment.signature -> Solana tx hash
// payment.status    -> "confirmed" within 2s
// Webhook fires to your URL with HMAC-SHA256 signature
